# 快速开始

本文档帮助你在本地启动 `cyop`。

## 前置条件

请先准备：

- **Bun** 1.2+
- **PostgreSQL**（推荐 Neon / 本地 Postgres）
- 一个可用的 **S3 兼容对象存储**（如果要测试素材上传）
- 可用的 AI 模型接口（如果要测试 Caption）

## 1. 安装依赖

```bash
bun install
```

## 2. 准备环境变量

复制示例文件：

```bash
cp apps/server/.env.example apps/server/.env
cp apps/web/.env.example apps/web/.env
```

### `apps/server/.env`

最基础配置：

```bash
DATABASE_URL=
BETTER_AUTH_SECRET=
BETTER_AUTH_URL=http://localhost:3000
CORS_ORIGIN=http://localhost:3001
```

上传相关：

```bash
S3_ACCESS_KEY_ID=
S3_SECRET_ACCESS_KEY=
S3_BUCKET=
S3_REGION=
S3_ENDPOINT=
S3_FORCE_PATH_STYLE=false
ASSET_PUBLIC_URL=
```

Caption 相关：

```bash
AI_CAPTION_MODEL=
AI_CAPTION_API_KEY=
AI_CAPTION_BASE_URL=
AI_CAPTION_PROMPT=
```

自动化事件（可选）：

```bash
AUTOMATION_WEBHOOK_URL=
```

### `apps/web/.env`

```bash
VITE_SERVER_URL=http://localhost:3000
```

## 3. 初始化数据库

```bash
bun run db:push
```

查看数据库：

```bash
bun run db:studio
```

## 4. 启动开发环境

```bash
bun run dev
```

默认端口：

- Web: `http://localhost:3001`
- API: `http://localhost:3000`
- Docs: `http://localhost:4000`

## 单独启动应用

```bash
bun run dev:web
bun run dev:server
bun --cwd apps/fumadocs run dev
```

## 常用检查命令

```bash
bun run check
bun run check-types
```

## 第一次联调建议

推荐按以下顺序验证：

1. 打开 `http://localhost:3000`，确认返回 `OK`
2. 打开 `http://localhost:3001`，确认首页可访问
3. 注册一个用户并登录
4. 在控制塔创建一个需求和数据集
5. 在素材库尝试上传一张图片
6. 在 AI 模型页配置模型并尝试生成 Caption
7. 在审核台检查生成结果

## 常见问题

### 数据库连接失败

检查：

- `DATABASE_URL` 是否正确
- 数据库实例是否可访问
- 是否已经执行 `bun run db:push`

### 登录后没有会话

检查：

- `BETTER_AUTH_URL`
- `CORS_ORIGIN`
- Web 是否使用正确的 `VITE_SERVER_URL`
- 本地环境下 Cookie / SameSite / HTTPS 配置是否与你的域名方案一致

### 上传失败

检查：

- S3 凭证与 Bucket 是否正确
- `S3_ENDPOINT` 是否需要 path-style
- `ASSET_PUBLIC_URL` 是否与对象可公开访问路径一致

### Caption 调用失败

检查：

- `AI_CAPTION_API_KEY`
- `AI_CAPTION_MODEL`
- `AI_CAPTION_BASE_URL`
- 模型 provider 是否与接口兼容
