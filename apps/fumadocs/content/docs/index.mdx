# cyop 文档

`cyop` 是一个围绕 **图像资产管理、AI Caption 自动化、人工审核与任务调度** 构建的全栈平台。

它把原本分散在需求文档、素材盘、脚本工具、审核后台里的流程，统一到一个可演进的产品和数据模型里。

## 核心能力

### 1. 需求建模

将业务目标、负责人、团队、优先级、目标覆盖率统一结构化存储。

### 2. 数据集运营

以数据集为中枢管理图片数量、已处理数量、AI 覆盖率、标签覆盖率与复核率。

### 3. 对象存储上传

通过服务端签名支持素材直传 S3 兼容对象存储，并在平台内完成登记。

### 4. Caption 生成与审核

支持：

- 模型注册
- Prompt 模板管理
- 单图生成
- 批量入队与处理
- 审核台人工通过 / 驳回 / 导出

### 5. 自动化编排

以任务模型记录批处理状态、进度、失败原因，并可推送 webhook 给外部系统。

## 仓库结构速览

```txt
apps/
├─ web        # React 控制台
├─ server     # Hono API
└─ fumadocs   # 文档站

packages/
├─ api        # tRPC routers + services
├─ auth       # Better Auth
├─ db         # Drizzle schema
├─ ui         # 共享组件
└─ config     # TS / Biome 配置
```

## 代码入口

| 路径 | 用途 |
| --- | --- |
| `apps/web/src/routes` | 页面与路由入口 |
| `packages/api/src/routers` | 后端业务入口 |
| `packages/api/src/services` | Caption / Storage / Automation 服务 |
| `packages/db/src/schema/platform.ts` | 平台业务数据模型 |
| `apps/server/src/index.ts` | Hono 服务入口 |
